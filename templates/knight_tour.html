The Knight's Tour Problem

Main menu where the user can access the linked list.

In this linked list, the user can store any number of initial positions.

The chessboard object stored in memory is an 8x8 array. In this case, the user selects as initial position the second row, and the second column.

An new node object is created and appended to the linked list object. The user can add more initial position or go back the main menu.

Next, the user can solve the Knight's Tour Problem for each node contained in the linked list.

The first 32 moves of the knight objet are performed using Wardnsdoffâ€™s heuristic. From his current position, the knight will always move to the postion were he will have the minimum options for the next position. We can see that that the knight moves along the inner border of the chessboard.

The last 32 moves of the knight objet are performed using backtracking. Indeed, the knight object is using his personal stack object to keep track of his path. When the knight is stuck, he will come back to the first path available and try to solve the tour. In worst case, he will attempt every single path possible to solve the tour. Lastly, we can notice that the behavior of the knight change drastically from the heuristic phase. From the intermediate position (1,3), he crosses the chessbaord following a diagonal. After four moves, the knight ends up at position (8,8).

Finally, the knight solves the problem and from initial position (2,2), he ends up in final position (5,8).
